# スキル作成スキル (skill_creator)

## 概要
繰り返し発生するタスクパターンを分析し、再利用可能なスキルとして定義・生成するスキルです。

## クイックスタート

### 1. スキル化したいタスクを特定
同じような指示を3回以上繰り返していませんか？

### 2. dashboard.mdにタスク指示を記載
```markdown
## 現在のタスク

### メインエージェント → サブエージェント

**タスクステータス**: スキル作成依頼

**担当サブエージェント**: skill_creator

**タスク内容**:
{スキル化したいタスクの説明}

**繰り返し回数**: {回数}

**具体例**:
1. {具体例1}
2. {具体例2}
3. {具体例3}

**スキル名候補**: {希望するスキル名（任意）}
```

### 3. skill_creatorエージェントを起動
メインエージェントがTask toolで呼び出し:
```
Task tool: skill_creator エージェント起動
```

### 4. 結果確認
dashboard.mdの「サブエージェント → メインエージェント」セクションで結果を確認

### 5. 生成されたスキルを使用
```
作成されたスキル: skills/{skill-name}/SKILL.md
使用方法: SKILL.md参照
```

## スキル化の判断基準

### ✅ スキル化すべき
- 同じ指示を3回以上繰り返している
- 定型的な作業フローがある
- 複数ステップで構成される

### ❌ スキル化不要
- 1回限りのタスク
- 1コマンドで完了する単純なタスク
- 頻繁に変更される可能性が高い

## 例: 品質チェックスキルの作成

### Before（スキル化前）
毎回手動で実行:
```
「typecheck、lint、testを実行してください」
「すべてのチェックをお願いします」
「動作確認をしてください」
```

### After（スキル化後）
```
「/quality_check を実行してください」
```

または

```
Task tool: quality_check スキル起動
```

## トラブルシューティング

### Q: スキル化すべきか判断できない
A: 以下を確認:
- 繰り返し回数は3回以上か？
- 複数ステップで構成されているか?
- 定型的なフローがあるか？

いずれか1つでも該当すればスキル化を検討してください。

### Q: 既存のスキルと似ている
A: skill_creatorが既存スキルとの統合を提案します。差異が小さい場合は統合、大きい場合は別スキルとして作成します。

### Q: スキル作成後の使い方がわからない
A: 生成された `skills/{skill-name}/SKILL.md` に詳細な使用方法が記載されています。
